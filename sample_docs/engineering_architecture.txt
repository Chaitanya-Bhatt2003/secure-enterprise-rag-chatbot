ACME CORPORATION — PLATFORM ARCHITECTURE OVERVIEW (INTERNAL)
Owner: Engineering | Classification: Internal

SYSTEM OVERVIEW
The Acme platform is a multi-tenant SaaS composed of four main services:
the API Gateway, the Core Application Service, the Search Service, and
the Async Worker fleet. All services run on Kubernetes (EKS) across three
availability zones. Terraform manages all infrastructure as code.

API GATEWAY
Kong-based gateway handling authentication (JWT via Auth0), rate limiting
(1,000 requests/minute per tenant on the standard tier), and request
routing. TLS 1.3 is enforced end to end; certificates rotate every 60
days via cert-manager.

CORE APPLICATION SERVICE
A Python/FastAPI monolith-in-transition. Bounded contexts (billing,
notifications) are being extracted into services. PostgreSQL 15 is the
primary datastore with read replicas in each AZ; row-level security
enforces tenant isolation. Redis handles session cache and rate-limit
counters.

SEARCH SERVICE
Hybrid retrieval over OpenSearch: BM25 keyword scoring fused with dense
vector similarity (768-dim embeddings, HNSW index). Index rebuilds run
nightly; incremental updates stream through Kafka with a 30-second SLA.

DEPLOYMENT PIPELINE
GitHub Actions builds and tests every PR; merges to main trigger a canary
deploy to 5% of traffic for 30 minutes before full rollout. Rollbacks are
one-click via Argo CD. Database migrations use expand-and-contract; no
destructive migration may ship in the same release that stops writing the
old column.

ON-CALL AND INCIDENTS
PagerDuty rotation is weekly per team. SEV-1 requires a 15-minute
acknowledgment SLA and a blameless postmortem within 5 business days.
The error budget is 0.1% monthly downtime (SLO 99.9%).
